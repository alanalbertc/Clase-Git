Lista de comandos comunes en git

Ver el historial de commits de un archivo
    git log "nombre-archivo"
    git log "nombre-archivo" --stat     //Muestra los cambios hechos en cada commit

Ver el historial de un archivo

    git show "nombre-archivo"
    A diferencia de git log, este comando muestra las líneas que sufrieron algún cambio y no solo  el título de los commit

Comparar dos versiones del mismo archivo

    git diff "ID-commit-original" "ID-commit-nuevo"

Viajes en el tiempo

    para volver de una forma brusca a una versión de un archivo se usa el comando 
        git reset "ID-commit" --hard    (Elimina el historial de commits, elimina lo que haya en staging, elimina todo y regresa a la versión del ID indicado)
    Para volver de una forma más amigable a una versión de un archivo se usa el comando 
        git reset "ID-commit" --soft  (Elimina el historial de commits pero mantiene los cambios guardados en staging)
    Para quitar un archivo del Staging (no lo borra, solo lo pasa al estado "Unstaged")
        git reset HEAD "nombre"

Creación de rama    
    git branch "nombre_rama"

Para cambiar de rama
    git checkout "nombre_rama"    
    **IMPORTANTE**
    Antes de cambiar de rama hay que asegurarse de que los cambios fueron guardados mediante un commit

Consultar las ramas existentes
    git branch

Unir dos ramas
    1. Irse a la rama en la que se van a traer los commit de la otra rama (normalmente master)
    2. git merge "nombre_rama"

Resolución de conflictos
    Cuando dos programadores han modificado las mismas líneas de código en el archivo, git nos dirá que hay un conflicto, hay que arreglarlo y después volver a realizar el merge

    Forma de arreglarlo.
    En Visual Studio Code nos aparecerá un mensaje en las líneas de conflicto junto con las opciones "conservar los cambios actuales", "aceptar los cambios entrantes" o "conservar ambos" en forma de link.

    La sintaxys de los mensajes de error es el siguiente

    <<<<<<<<< "rama_actual"
        "liena(s) con conflicto"
    ---------
    >>>>>>>>> "rama_entrante"
        "liena(s) con conflicto"
    
    Entonces simplemente queda seleccionar una de las opciones y VSC en automático eliminará las líneas con la sintaxys del error y dejará las líneas con conflicto con la opcion que hayamos seleccionado.
    
    La forma manual de realizar esto (por ejemplo en el block de notas) es eliminar las líneas correspondientes al error "<<< y >>>" y dejar las linea(s) con conflicto con la opción que querramos.


    Enviar un repositorio local a GitHub

    1.  git remote add origin "link_repo_en_GitHub" //agrega al repositorio remoto "origin master"
    2.  git remote  // Comprueba que tenemos un origin en donde hacer pull y push
        git remote -v // "De manera verbal" muestra el link del repositorio
    3.  git push origin main
    4.  Si aparece el error "falied to push some refs to..."
        Es porque no se sabe lo que se ha estado trabajando en el repo origin, entonces primero hay que traer su contenido con un pull

        git pull origin master
    Si aparece fatal: refusing to merge unrelated histories

    git pull origin master --allow-unrelated-histories  //forzar la fusión de las historias distintas

    Si se desea eliminar un repositorio remoto entonces se ejecuta el siguiente comando
        git remote rm "origin_name"
